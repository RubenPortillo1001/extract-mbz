1. Configuración Peligrosa del Servidor

Opciones de configuración del servidor SQL detectadas como activas que expanden la superficie de ataque. Fuente: ServerConfigAudit.

🔴 xp_cmdshell = 1 (HABILITADO)
Permite ejecutar comandos arbitrarios del sistema operativo Windows directamente desde T-SQL. Cualquier usuario con permisos EXECUTE sobre xp_cmdshell puede tomar control total del servidor. Vector de escalada de privilegios máxima — combinado con los db_owner detectados, el riesgo es inmediato.
CM-6: Configuration Settings CM-7: Least Functionality AC-3: Access Enforcement
🔴 Cuenta SA habilitada
La cuenta "sa" (System Administrator) está activa. Es el objetivo primario de ataques de fuerza bruta automatizados. Tiene privilegios irrestrictos sobre toda la instancia SQL y no puede ser auditada con granularidad. Fue creada en 2003 (hace 23 años) y no tiene expiración de contraseña.
AC-2: Account Management IA-5: Authenticator Mgmt IA-4: Identifier Management
🟠 CLR Enabled = 1 (HABILITADO)
Permite cargar ensamblados .NET externos dentro del proceso SQL Server. Un atacante con acceso suficiente puede subir una librería maliciosa compilada para ejecutar código arbitrario con los privilegios del servicio SQL Server.
CM-7: Least Functionality SI-3: Malicious Code Protection
🟠 Remote Access = 1 (HABILITADO)
Habilita conexiones remotas server-to-server (linked servers RPC). Amplía la superficie de ataque y puede ser utilizada para movimiento lateral dentro de la red interna de la CSJ.
AC-17: Remote Access SC-7: Boundary Protection

6. Plan de Remediación Priorizado

FASE 1 — Acción Inmediata (P1): Dentro de 48 horas

1
Deshabilitar xp_cmdshell
Es el riesgo individual más grave. Ejecutar en instancia GUATAMBU como sysadmin.
-- Deshabilitar xp_cmdshell
EXEC sp_configure 'show advanced options', 1; RECONFIGURE;
EXEC sp_configure 'xp_cmdshell', 0; RECONFIGURE;
EXEC sp_configure 'show advanced options', 0; RECONFIGURE;
2
Deshabilitar CLR y Remote Access
Reducir superficie de ataque del servidor.
EXEC sp_configure 'show advanced options', 1; RECONFIGURE;
EXEC sp_configure 'clr enabled', 0; RECONFIGURE;
EXEC sp_configure 'remote access', 0; RECONFIGURE;
EXEC sp_configure 'show advanced options', 0; RECONFIGURE;
3
Deshabilitar y cambiar contraseña de la cuenta SA
Aplicar inmediatamente. La cuenta SA nunca debe estar activa en producción.
-- Cambiar contraseña a valor complejo aleatorio primero, luego deshabilitar
ALTER LOGIN [sa] WITH PASSWORD = '<contraseña_aleatoria_32_chars>';
ALTER LOGIN [sa] DISABLE;
4
Investigar la cuenta d_lopez (creada 2026-04-06)
Cuenta creada 1 día antes de la auditoría con db_owner en Consulres. Posible acceso no autorizado. Revocar db_owner y revisar quién la creó.
-- Auditar quién creó d_lopez y cuándo
USE Consulres;
SELECT name, create_date, modify_date, default_schema_name
FROM sys.database_principals WHERE name = 'd_lopez';

-- Revocar db_owner inmediatamente
EXEC sp_droprolemember 'db_owner', 'd_lopez';
5
Eliminar usuarios huérfanos
Ejecutar en las dos BDs afectadas.
-- ContenedorDLL: revocar db_owner del usuario huérfano, luego eliminar
USE ContenedorDLL;
EXEC sp_droprolemember 'db_owner', 'D_Lopez';
DROP USER [D_Lopez];

-- Sig_pub_reg
USE Sig_pub_reg;
DROP USER [UserSigPub];

FASE 2 — Acción de Corto Plazo (P2): Dentro de 30 días

6
Revocar sysadmin de cuentas personales (m_peralta, s_villa, m_silvera, CSJ\appm1)
Consultar con DAI qué nivel de acceso realmente necesitan. Asignar roles mínimos.
-- Ejemplo para m_peralta (repetir para cada login)
ALTER SERVER ROLE [sysadmin] DROP MEMBER [m_peralta];
-- Asignar rol específico si es DBA: dbcreator o securityadmin, nunca sysadmin
7
Revocar db_owner de todos los usuarios de bases de datos de producción
Reemplazar con db_datareader + db_datawriter para usuarios operativos. Para cuentas de aplicación (usersigjur, conexion_lederes), otorgar solo los permisos que la aplicación realmente usa. 
-- Plantilla — repetir para cada BD y usuario
USE Consulres;
EXEC sp_droprolemember 'db_owner', 's_insfran';
EXEC sp_addrolemember  'db_datareader', 's_insfran';
-- Agregar db_datawriter solo si el usuario necesita escribir
8
Habilitar expiración de contraseñas en todos los logins SQL
11 logins carecen de expiración. Establecer período máximo de 90 días.
-- Verificar logins sin expiración
SELECT name, is_expiration_checked, is_policy_checked
FROM sys.sql_logins
WHERE is_expiration_checked = 0 AND is_disabled = 0;

-- Habilitar expiración (ejemplo)
ALTER LOGIN [user_resolucioncon] WITH CHECK_EXPIRATION = ON, CHECK_POLICY = ON;
9
Revisar y justificar acceso de g_martinez en 9 bases de datos
El usuario g_martinez tiene db_backupoperator en 9 BDs. Si necesita hacer backups centralizados, documentar la justificación. Si no, revocar acceso en las BDs no relacionadas con su función.

FASE 3 — Mejora Continua (P3): Dentro de 90 días

10
Implementar política formal de gestión de cuentas
Definir proceso: quién aprueba creación de logins, revisión semestral de accesos, proceso de desvinculación de personal que incluya revocación SQL inmediata. Alinear con NIST AC-2 (Account Management Policy).
11
Configurar SQL Server Audit para eventos críticos
Auditar: LOGIN_FAILED, ADD_MEMBER_TO_DB_ROLE, ALTER_SERVER_CONFIGURATION, EXEC_xp_cmdshell. Enviar logs a sistema centralizado (SIEM). Alinear con NIST AU-2, AU-9.
12
Auditar otras instancias SQL en la red CSJ
Aplicar el mismo script de auditoría a todos los servidores SQL de la CSJ. Los patrones encontrados en GUATAMBU (db_owner masivo, SA activa, xp_cmdshell) probablemente se repiten en otros servidores.

Mensaje clave para la Dirección:
La combinación de xp_cmdshell habilitado + múltiples usuarios con sysadmin + cuenta SA activa significa que un único evento de compromiso de cualquiera de estas cuentas daría al atacante control total del sistema operativo Windows del servidor GUATAMBU, no solo de las bases de datos. Las configuraciones detectadas (xp_cmdshell, CLR) no son necesarias para la operación ordinaria de los sistemas de la CSJ y deben desactivarse sin excepción. La remediación de la Fase 1 (ítems 1-5) no requiere downtime prolongado y puede ejecutarse en una ventana de mantenimiento de 2 horas.
